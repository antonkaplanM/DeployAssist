🚀 Getting Started

Welcome to the Deployment Assistant! This application helps you manage and monitor Professional Services requests, track deployment progress, and maintain data quality through validation rules.

First Time Setup

  1. Navigate to Settings to configure Salesforce authentication
  2. Test connectivity to ensure all integrations are working
  3. Visit Validation Rules to configure data validation preferences
  4. Explore the Dashboard for an overview of your system status

📊 Dashboard

Your central hub for monitoring system health and validation status.

Data Validation Monitor

Real-time validation status for Professional Services records. Use the time frame selector (1d/1w/1m/1y) to adjust the monitoring period.

  • Green status: All records pass validation
  • Red status: Validation failures detected with details
  • Click "View Record" to navigate to specific PS records

API Status

Shows the current status of server connectivity and API responses.

Tech Stack

Information about the underlying technologies powering the application.

🔧 Provisioning Monitor

Monitor and manage Professional Services requests with advanced search and filtering capabilities.

Key Features:

  • 🔍 Search: Type-ahead search for Technical Team Requests and Accounts
  • 🎯 Filters: Filter by Request Type, Status, and other criteria
  • 📄 Pagination: Browse through large datasets efficiently
  • 📊 Data Validations: View validation status for each record
  • 📦 Product Details: Click product groups to view entitlement details
  • 📤 Export: Export filtered results for external analysis
  • 🔄 Refresh: Update data in real-time

Pro Tips:

  • • Use the search box for quick access to specific requests
  • • Click column headers to sort data
  • • Check the Data Validations column for compliance status

✅ Validation Rules

Configure and manage data validation rules applied to Professional Services requests.

Available Rules:

App Quantity Validation

Ensures app quantities are set to 1, except for IC-DATABRIDGE products which are always valid.

Model Count Validation

Validates that the number of models doesn't exceed 100 per request.

Entitlement Date Overlap Validation

Prevents overlapping date ranges for entitlements with the same product code.

How to Use:

  1. Toggle rules on/off using the switches next to each rule
  2. Click "Test Rules" to validate all current PS records
  3. View detailed results and error reports
  4. Settings automatically sync with the Dashboard validation monitor

📈 Analytics & Account History

View comprehensive analytics and track the complete history of Technical Team Requests for any account.

Analytics Overview:

  • Total Accounts: Number of active accounts in the system
  • Total Products: Count of provisioned products
  • Average Products per Account: Distribution metrics
  • Account Overview Table: Detailed breakdown by account

Account History Feature

Track the complete evolution of product entitlements for any account through a chronological view of all Technical Team Requests.

🔍 Smart Search

Search by either:

  • Account Name: e.g., "Bank of America" - shows all requests for that account
  • PS-ID: e.g., "PS-4331" - automatically loads the associated account's history
  • Auto-detection identifies your search type automatically
  • Real-time autocomplete suggestions as you type

⚙️ Configurable Display

  • Default Limit: Shows latest 5 requests (fastest loading)
  • Options: Latest 5, 10, 25, 50, or All Requests
  • Count Indicator: Always shows "X of Y requests" for clarity
  • Chronological Order: Most recent requests appear first

📦 Product Information

  • Collapsible Categories: Models, Data, and Apps are collapsed by default
  • Click to Expand: Click any product category to view details
  • Product Modal: Click category buttons to open detailed modal with full product list
  • Expandable Details: Expand each request row for complete entitlement information

🔄 Product Change Comparison

Enable "Show Product Changes" to see:

  • Added Products: Highlighted in green - new products in this request
  • Removed Products: Highlighted in red - products removed from previous request
  • Chronological Comparison: Each request compared to its previous version
  • Evolution Tracking: See how product sets changed over time

⚡ Actions & Navigation

  • Actions Dropdown: Click the three-dot menu (⋮) for options
  • View in Provisioning Monitor: Navigate to see the request with exact match filtering
  • Exact Search: PS-89 shows only PS-89, not PS-898 or similar IDs
  • Seamless Integration: Switch between Account History and Provisioning Monitor easily

💡 Pro Tips:

  • Start with the default limit of 5 for faster loading, then expand as needed
  • Use "Show Product Changes" to quickly identify configuration changes
  • Click product categories to open modals with detailed entitlement information
  • Use the Actions dropdown to navigate to Provisioning Monitor for related records
  • Expand request rows to see complete technical details and metadata
  • Click the "Clear Selection" button to start a new search

🔧 Troubleshooting

Common Issues & Solutions

🔌 "Unable to load validation data" Error

Solution: Check Salesforce authentication in Settings. Ensure your credentials are valid and you have access to Prof_Services_Request__c object.

📊 Empty Dashboard or Missing Data

Solution: Verify time frame selection and ensure PS records exist in the selected period. Check validation rules are enabled.

🔍 Search Not Working

Solution: Ensure search terms are at least 2 characters. Clear browser cache if results seem outdated.

🗺️ Roadmap Not Loading

Solution: Check Atlassian API configuration and ensure assignee names are spelled correctly (e.g., "Kevin Yu").

Performance Tips

  • • Use specific time frames for faster validation monitoring
  • • Apply filters before searching large datasets
  • • Refresh data periodically for latest information
  • • Use browser refresh if UI becomes unresponsive

Need More Help?

If you continue experiencing issues, check the browser console for detailed error messages or contact your system administrator.
