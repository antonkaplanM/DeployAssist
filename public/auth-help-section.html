The application includes a secure authentication system with role-based access control (RBAC) to manage user access and permissions.

Key Features

  • User Management: Create, edit, and delete user accounts
  • Role-Based Access Control: Assign roles to control what users can access
  • Secure Sessions: JWT-based authentication with automatic timeouts
  • Password Security: Bcrypt hashing, complexity validation, account lockout after 5 failed attempts
  • Remember Me: Optional 30-day session for convenience
  • Audit Logging: Track role changes and security events

Logging In

  1. Navigate to the application URL (redirects to login if not authenticated)
  2. Enter your username and password
  3. Optionally check "Remember me" for extended session (30 days)
  4. Click Sign In

Default Admin: Username is admin (password set by administrator)

Session Management

Session Duration: Maximum 24 hours

Inactivity Timeout: 60 minutes of inactivity logs you out automatically

Remember Me: Extends session to 30 days (check box at login)

Logout: Click the Logout button in the sidebar to end your session

User Management (Admin Only)

Administrators can manage users and roles through the User Management interface.

  • Access: Click "User Management" in the sidebar (admin only)
  • Create Users: Click "+ Create User", fill in details, assign roles
  • Edit Users: Click "Edit" button next to any user
  • Change Passwords: Click "Password" button (admins can change any user's password)
  • Activate/Deactivate: Toggle user status when editing
  • Delete Users: Remove users permanently (cannot delete yourself)

Role Management (Admin Only)

Create custom roles to organize access permissions for different user types.

  • System Roles: admin (full access), user (standard access)
  • Create Custom Roles: Click "+ Create Role" in User Management
  • Assign Roles: Select roles when creating/editing users
  • Delete Roles: Custom roles can be deleted if not assigned to users
  • Protection: System roles cannot be deleted

Password Requirements

  • Minimum 8 characters
  • Follows Gmail-standard security practices
  • Account locks after 5 failed login attempts (15-minute lockout)

Changing Your Password

Any user can change their own password:

  1. Log in to your account
  2. Go to User Management (or ask admin)
  3. Admin can reset any password using the "Password" button
  4. Enter new password (minimum 8 characters)
  5. Confirm the change

After 5 failed login attempts, your account is locked for 15 minutes to prevent unauthorized access.

Solutions:

  • Wait 15 minutes and the lockout automatically expires
  • Contact administrator to unlock immediately using the unlock script

Permission Levels

Admin Role:

  • Full access to all features
  • User Management access
  • Role Management access
  • Can change any user's password
  • Can create, edit, delete users
  • Can create and delete custom roles

User Role:

  • Access to all application features
  • Dashboard, Analytics, Provisioning, Expiration monitoring
  • Cannot access User Management
  • Cannot manage roles
  • Can change own password (via admin)

Custom Roles:

  • Created by admins for specific access needs
  • Currently inherit user-level permissions
  • Designed for future granular permission assignment

üí° Best Practices

  • Change default admin password immediately after first login
  • Use strong, unique passwords for each user
  • Log out when leaving your workstation unattended
  • Only assign admin role to users who need user management access
  • Regularly review user accounts and deactivate unused accounts
  • Create custom roles for different access levels as needed
  • Don't share passwords between users

Security Features

  • Password Hashing: Bcrypt algorithm with salt
  • HTTP-Only Cookies: Protects session tokens from JavaScript access
  • Rate Limiting: Prevents brute-force attacks on login endpoint
  • Account Lockout: Automatic lockout after failed attempts
  • Session Expiration: Automatic logout on inactivity
  • Audit Trail: Logs all role changes and security events
  • CSRF Protection: Secure cookie-based authentication
